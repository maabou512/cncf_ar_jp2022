いらっしゃいませ！思い出に残る年になりました

2022 年を振り返ると、私たちが共に乗り越え、達成したことを誇りに思います。私たちが直面し、取り組み続けている集合的な課題にもかかわらず、クラウド ネイティブ エコシステムは急増しています。私たちは新しい産業やプロジェクトを歓迎し、コミュニティをデンマークの人口を超える 710 万人のクラウド ネイティブ デベロッパーにまで拡大しました。

クラウド ネイティブはキャズムを越え、これまで以上に多くの組織がソフトウェア企業になりつつあります。これは、KubeCon + CloudNativeCon イベントに反映されており、メルセデス ベンツやボーイングなどの企業が、有名なテクノロジー ブランドと並んで基調講演を行いました。

もちろん、これは真にグローバルで歓迎的な実行者のコミュニティである #TeamCloudNative なしでは不可能です。17 万 8,000 人を超える皆様の献身と貢献に個人的に感謝したいと思います。大小を問わず、私たちはこの旅路を共に歩んでいます。

私たちのマイルストーンについてお読みになり、今年一緒に達成した印象的な進歩を振り返っていただければ幸いです。

エンド ユーザー コミュニティ

Cloud Native Computing Foundation (CNCF) 内のエンド ユーザーは、クラウド ネイティブ テクノロジを内部で使用し、クラウド ネイティブ サービスを外部に販売しないメンバー企業であり、ベンダー、コンサルタント会社、トレーニング パートナー、または電気通信会社ではありません。エンド ユーザー企業内の個人クラウド ネイティブ アーキテクチャを使用して問題を解決し、より包括的で反復的なプロセスを作成するセルフサービス ソリューションをチームに提供することに情熱を持っています。

CTOサミット

チームがクラウド ネイティブ戦略を採用するにつれて、組織が直面する課題に対する「万能の」ソリューションがないことは明らかです。そのため、CNCF は 2022 年に CTO サミットを立ち上げました。チャタム ハウスが管理するこのサミットでは、エンド ユーザー企業の CTO が招集され、組織が人、プロセス、テクノロジーを活用してクラウド ネイティブ コンピューティングの回復力を向上させる方法について話し合います。

Boeing、Fidelity、Intuit などのエンド ユーザーが集まり、組織内で技術的な課題にどのように対処しているかについて非公開で会話できる CTO サミットを開始しました。多くの場合、最も影響力のある教訓は仲間から得られます。

新しい CNCF.io

5 月に、再設計および再構築された CNCF.io を開始しました。このサイトは、より効果的で活気に満ちたインタラクティブな方法でコミュニティを表現することに重点を置いています。CNCF とエンド ユーザーのコンテンツ、イベント、ニュース、ブログ記事を共有するための追加の領域を提供します。また、動的なメガ メニューと簡素化されたナビゲーションのおかげで、UX とナビゲーションが改善されています。重要なのは、アクセシビリティを倍増させていることです。Web コンテンツ アクセシビリティ ガイドライン (WCAG) 2.1 に従い、CNCF.io でレベル AA のアクセシビリティを目指しています。年間を通して、コントリビューターを含む CNCF サブサイトに新しいデザインを展開し、すべてのサイトに WCAG ガイドラインを適用しました。

KCDエボリューション

また、プログラムの成功を確実にするために、KCD コミュニティと主催者をサポートするための追加の対策を実施しました。

  • KCD GitHub リポジトリのオーガナイザー利用規約が更新され、インクルーシブ オープン ソース コミュニティ オリエンテーションを受講するというオーガナイザーの既存の要件に加えて、新しい条項が追加されました。新しい用語は次のとおりです。
    • 最終的なプログラム スケジュールが多様であることを確認することに同意します (たとえば、すべてのスピーカーが 1 つの性別、1 つの文化、または年齢であるとは限りません)。
    • 多様なラインナップを確保するために、公開前にレビューと承認を受けるために、プログラムのコピーを kcd@cncf.io に送信する必要があります。
    • 主催者は、講演者のラインナップが代表する企業の点で多様であることを保証します。
  • KCD の主催者は、非技術的かつ非コード的な貢献である講演を検討し、含めることをお勧めします。
  • 7 月に、計画プロセス中に主催者をガイドするのに役立つトピックを含む、1 時間の毎月の主催者会議の開催を開始しました。8月のテーマは「多様なラインナップをキュレートする方法」。この会議では、スケジュール内で多様性を構築し、採用する方法について、主催者向けのリソースと推奨事項が取り上げられました。スライドを提供し、セッションを記録します。

行動規範プロセスの改善

対面イベントの開催に戻ったとき、行動規範インシデントの数と深刻度の両方が増加しました。行動規範への関心が高まったことで、以前は完全に財団が管理していたプロセスを、コミュニティの代表者を含めて透明性を高めることで近代化する必要性が浮き彫りになりました。その結果、今年は次のようになりました。

  • 私たちは行動規範ワーキンググループを立ち上げ、最新の構造と一連の手順を開発しました。彼らの仕事は進行中であり、コミュニティの誰でも参加して貢献することができます。ワーキング グループのリポジトリにアクセスして、参加し、進捗状況を確認し、詳細を確認してください。
  • また、コミュニティ メンバーとスタッフの両方で構成される暫定行動規範委員会を立ち上げ、インシデントを裁定する一方で、より恒久的な構造がワーキング グループによって開発中です。委員会が運営しているインシデント解決手順は、今年初めに公開され、パブリック コメント期間を経て最終決定されました。

トレーニングと認定

3/6

CNCF は、2022 年にクラウド ネイティブ エコシステムを成長させるというコミットメントを強化しました。世界的に認められた認定資格を拡大し、雇用機会を増やし、より多くの人々がクラウド ネイティブ テクノロジーの実用的なアプリケーションのスキルアップを支援します。

今年、私たちは Prometheus 認定アソシエイト (PCA) を開始しました。これは、オープンソースの監視およびアラート ツールキットである Prometheus を使用して、オブザーバビリティとスキルに関する基礎知識を証明し、エンジニアリングする認定資格です。これらにより、学習者は自分の経験に関連する特定のトピックに関する資格を取得できます。最初の SkillCreds の 2 つには、Developing Helm Charts (SC104) と Open Data Formats: YAML (SC101) が含まれます。これらのマイクロクレデンシャルは短く、トピック固有で、費用対効果が高いです。トレーニングと認定の重要性、およびCNCFコースが開催されるという点は、外国人が北京のビザを申請できる中国で今年強調されました。彼らは CKA 認定を取得します。

安全

CNCF は、Open Source Technology Improvement Fund (OSTIF) との戦略的パートナーシップにより、2022 年を通じて多数のオープンソース セキュリティ監査を実施しました。多くのプロジェクトがセキュリティ監査を完了し、132 件のセキュリティ修正と改善、45 件の CVE の修正、51 件のセキュリティ ツールの構築が行われました。また、2023 年に開始される初の CloudNative SecurityCon も発表しました。これにより、アプリケーション開発者と最新のセキュリティ専門家が、以前のものを徐々に改善するソリューションを提案するだけでなく、最先端のプロジェクトと最新のセキュリティ アプローチの進歩に余地を与えることができます。さらに、CNCF はさまざまなプロジェクトのファジング監査に資金を提供し、その結果、何百ものバグが発見されました。
